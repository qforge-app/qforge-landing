▶ 1 — Environment Setup Prerequisites & Config

Prerequisites

  • ✓ VS Code 1.85+ with QForge extension installed (package: qforge-4.0.0.vsix)
  • ✓ Python 3.11+ with venv at C:/qforge/qforge-backend/venv/
  • ✓ Node.js 18+ and npm for extension build verification
  • ✓ MongoDB running locally on localhost:27017 (or Atlas URI in env)
  • ✓ Git — to verify commit hashes in Known Issues
  • ✓ Redis (optional) — required for TC-SEC-005 rate limiter test; set REDIS_URL

Activate Backend & Run Automated Tests

# 1. Activate venv (Windows)
C:/qforge/qforge-backend/venv/Scripts/activate

# 2. Install dependencies
pip install -r C:/qforge/qforge-backend/requirements.txt

# 3. Copy and fill env
cp C:/qforge/qforge-backend/.env.example C:/qforge/qforge-backend/.env

# 4. Start backend
cd C:/qforge/qforge-backend
uvicorn main:app --reload --port 8000

# 5. Run automated test suite (separate terminal, venv active)
cd C:/qforge/qforge-backend
pytest tests/ -v

▶ Suite 9 — Chat Execution Sandbox ⚡ 8 test cases — Industry Break #1

Validates services/chat_sandbox.py end-to-end via POST /api/chat. TC-SBX-008 Security — Blocked Pattern in Generated Code
Purpose
Validates that _pre_validate() inside execute_circuit() blocks code containing dangerous patterns before any exec() call. Sandbox must not execute code that contains os., subprocess, open(, or __subclasses__.
Steps
Call run_chat_code_blocks() directly with a markdown response containing each blocked pattern. Run 4 sub-cases.
from services.chat_sandbox import run_chat_code_blocks
import asyncio

# Sub-case A: os.system
resp_a = "```python\nimport os\nos.system('rm -rf /')\nfrom qiskit import QuantumCircuit\nqc = QuantumCircuit(1)\n```"
# Sub-case B: subprocess
resp_b = "```python\nimport subprocess\nsubprocess.run(['whoami'])\nfrom qiskit import QuantumCircuit\nqc = QuantumCircuit(1)\n```"
# Sub-case C: open()
resp_c = "```python\nopen('/etc/passwd').read()\nfrom qiskit import QuantumCircuit\nqc = QuantumCircuit(1)\n```"
# Sub-case D: __subclasses__ escape
resp_d = "```python\n().__class__.__bases__[0].__subclasses__()\nfrom qiskit import QuantumCircuit\nqc = QuantumCircuit(1)\n```"

for label, resp in [('A:os',resp_a),('B:subprocess',resp_b),('C:open',resp_c),('D:subclasses',resp_d)]:
    r = asyncio.run(run_chat_code_blocks(resp))
    print(f"{label} | flagged={r[0]['flagged']} | error={r[0].get('error','')[:60]}")
Acceptance Criteria
All 4 sub-cases: flagged = true  |  All 4 sub-cases: error contains "Blocked pattern"  |  All 4 sub-cases: counts = {} (no execution happened)  |  No host OS commands executed  |  No file system access occurred  |  Backend remains healthy (GET /health returns 200) after all sub-cases

from qiskit_aer import AerSimulator

# BV for hidden string s = 1011
# 4 query qubits (q0-q3) + 1 ancilla (q4)
secret = "1011"
n = len(secret)
qc = QuantumCircuit(n + 1, n)

# Ancilla in |-> state
qc.x(n)
qc.h(n)

# Hadamard on query
qc.h(range(n))

# Oracle: CNOT on ancilla for each bit=1 in secret
for i, bit in enumerate(reversed(secret)):
    if bit == '1':
        qc.cx(i, n)

# Hadamard again to decode
qc.h(range(n))
qc.measure(range(n), range(n))

sim = AerSimulator()
counts = sim.run(qc, shots=1024).result().get_counts()
