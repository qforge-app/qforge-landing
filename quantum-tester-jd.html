About QForge

We're building the GitHub Copilot for quantum computing — a VS Code extension that helps developers write, debug, visualize, and certify quantum circuits. Our Circuit DNA fingerprinting system generates a four-layer biological fingerprint for every circuit (genotype, topology, sequence, phenotype), and our Proof-of-Quantum-Work certification is the world's first credential earned by executing real circuits — not by answering multiple-choice questions. We're pre-seed, live at qforge.app, and published on the VS Code Marketplace.

Our stack touches everything modern: TypeScript, FastAPI, MongoDB, Qiskit, Cirq, PennyLane, IBM Quantum hardware, and Anthropic/OpenAI LLMs. The problems are genuinely hard — quantum correctness, streaming AI responses, and tamper-proof certificate verification don't fit neatly into standard QA playbooks. That's exactly why we need someone who knows quantum computing and software quality in equal measure.

The Role

You'll be our first QA hire. Own quality end-to-end. Work directly with the founding team from day one. Every bug you find protects real quantum developers who are writing circuits, earning certifications, and building careers on this platform. You won't be maintaining a legacy QA process — you'll be inventing it from scratch in a category that didn't exist two years ago. This is a high-impact role in a startup that is actively defining what quantum software development looks like.

What You'll Do

1
Design and execute quantum circuit correctness tests — verify state vectors, probability distributions, and TVD scores against expected outputs.
2
Validate LLM-generated code: confirm it compiles and produces correct quantum states when executed on Qiskit Aer simulators.
3
Test Circuit DNA fingerprinting for hash determinism, layer accuracy (genotype, topology, sequence, phenotype), and similarity scoring consistency.
4
Verify the Proof-of-Quantum-Work certification pipeline end-to-end: submit assignment → grade → issue cert → employer verification API.
5
Execute security test suites: JWT tampering, injection attacks, CORS policy enforcement, rate limiting, and auth boundary testing.
6
Test the VS Code extension across Windows, Mac, and Linux — activation sequences, all 55 commands, 40 webview panels, and streaming AI responses.
7
Write and maintain automated regression tests using pytest; contribute to CI coverage so every merge is validated automatically.
8
Write detailed bug reports with quantum-specific context: expected state vectors or count distributions vs. actual outputs with reproduction steps.
9
Verify bug fixes before release — own the regression suite and be the final gate before any feature ships to production.
10
Collaborate with backend engineers to diagnose failures in LLM generation, quantum simulation accuracy, and async streaming endpoints.

You Must Have

  • Solid understanding of quantum computing fundamentals — superposition, entanglement, gates, measurement, and state vectors; you can reason about what a circuit should produce.
  • Hands-on experience with Qiskit, Cirq, or PennyLane — you should know what a Bell state looks like in code and how to verify it with statevector simulation.
  • Python proficiency — able to write pytest test cases, run Qiskit circuits programmatically, and interpret simulation results without guidance.
  • REST API testing experience using Postman, curl, or a similar tool; comfortable testing authenticated endpoints and parsing JSON responses.
  • Ability to interpret quantum probability distributions — you know that |00⟩ = 0.5, |11⟩ = 0.5 is the correct output of a Bell circuit, and you can flag deviations confidently.
  • Git basics — clone, commit, branch, and open a PR; can collaborate on a shared codebase without losing work.

Bonus If You Have

  • Academic background in quantum information, quantum physics, or quantum engineering (B.Tech, M.Tech, MSc, or PhD).
  • FastAPI or async Python experience — understanding of async/await, background tasks, and middleware helps when diagnosing backend failures.
  • VS Code extension development or testing experience — familiarity with the Extension API, webview panels, or the activation lifecycle.
  • LLM / AI product testing experience — you've tested prompt pipelines, model outputs, or streaming inference APIs before.
  • MongoDB or NoSQL querying — able to inspect collections directly to verify data integrity during certification and fingerprinting tests.
